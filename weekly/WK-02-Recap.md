# Week 2 Recap

This week, I built a synthetic telemetry pipeline for an Aido Rover fleet and analyzed it using methods inspired by quantitative finance panel data. The main output was a parameterized telemetry generator that can simulate multiple rover units over time with a reproducible random seed. The generated dataset is structured as a `unit_id × timestamp` panel, which is similar to an equity dataset structured as `stock_id × date`.

The telemetry generator includes key rover signals such as speed, CLI horizon, battery voltage, battery state of charge, GPS fix quality, RSSI, IMU acceleration and gyroscope channels, motor current for four drives, six joint angles, and task-success flags. I also added realistic missingness patterns, including RSSI dropouts, IMU block-level dropouts, motor-current sensor gaps, and occasional GPS-quality missingness. This made the dataset more realistic and gave the EDA notebook meaningful missing-data patterns to inspect.

The EDA notebook focused on stationarity, rolling metrics, cross-sectional ranking, missingness, and inter-unit correlation. The ADF tests showed that selected telemetry channels were not stationary in the sample, largely because the synthetic data includes daily operating cycles. I also computed 7-day rolling means and standard deviations for battery SoC and motor current. These rolling features helped smooth short-term noise and made the telemetry data easier to interpret, similar to moving averages and rolling volatility in financial data.

The most finance-like part of the analysis was the cross-sectional percentile ranking. At each timestamp, each rover was ranked against the rest of the fleet based on KPIs such as battery SoC, motor current, RSSI, and task success. This is directly analogous to ranking stocks by valuation, quality, momentum, or other factor metrics on the same date. The motor-current percentile heatmap showed frequent rank changes, while the battery-SoC percentile heatmap showed more persistent horizontal bands. This looked similar to equity factor data, where some factors are noisy and short-horizon while others are more persistent across time.

The inter-unit motor-current correlation heatmap was another strong analogy to equity data. Many rovers showed positive co-movement because they shared the same daily activity cycle, just as stocks in the same sector may co-move due to common market or sector exposure. However, each rover still had its own unit-specific variation from efficiency, motor resistance, battery health, and sensor noise. This made the telemetry panel behave like a realistic dataset with both common factors and idiosyncratic behavior.

Overall, this week connected robotics telemetry with quantitative finance workflows. The main lesson was that fleet telemetry can be treated like a panel dataset: each rover behaves like an asset, each timestamp behaves like a trading date, and each telemetry signal can be engineered into rolling, cross-sectional, or quality-control features.
